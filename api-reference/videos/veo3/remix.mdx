## 路径参数

<ParamField path="task_id" type="string" required>
  原视频任务 ID

  这是之前通过视频生成接口返回的 task\_id，原始视频任务状态必须为 `success`
</ParamField>

## Authorizations

<ParamField header="Authorization" type="string" required>
  所有 API 端点都需要 Bearer Token 认证

  获取您的 API Key：

  访问 [API Key 管理页面](https://XCompute.us/console/token) 以获取您的 API Key

  将其添加到请求头中：

  ```
  Authorization: Bearer YOUR_API_KEY
  ```
</ParamField>

## Body

<ParamField body="model" type="string" required>
  视频模型名称，必须与原视频使用的模型一致

  支持的模型：

  * `veo3.1-fast` - 快速版本
  * `veo3.1-quality` - 高质量版本

  示例：`"veo3.1-fast"`
</ParamField>

<ParamField body="prompt" type="string" required>
  续写提示词，描述延长部分的内容
</ParamField>

<ParamField body="raw" type="boolean">
  是否只返回延长部分的视频

  * `true` - 只返回延长后的视频

  默认值：`false`
</ParamField>

<ParamField body="aspect_ratio" type="string">
  延长视频的比例

  支持的值：

  * `16:9`
  * `9:16`
</ParamField>

<ParamField body="resolution" type="string">
  视频分辨率

  支持的值：

  * `720p`（默认）
  * `1080p`
  * `4k`
</ParamField>

## 注意事项

<Note>
  **model 必须一致**：延长时的 model 必须与生成原始视频时使用的模型相同。用 `veo3.1-fast` 生成的视频，延长时也必须填 `veo3.1-fast`。
</Note>

<Note>
  **仅支持已完成的视频**：原始视频任务状态必须为 `success`。
</Note>

<Note>
  **URL 中的 task\_id**：是第一步生成视频时返回的 `task_id`（非上游 ID）。
</Note>
